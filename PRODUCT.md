# Product

## Register

product

## Users

Manuscript Forge is used by authors, worldbuilders, game masters, and editors who want to turn Markdown drafts into a polished fantasy manuscript PDF without doing layout work by hand. They are usually testing longer prose, changing source text, adjusting visual assets, and exporting again.

## Product Purpose

The product accepts pasted Markdown or an uploaded `.md` file, plans the manuscript structure with OpenAI, generates a small number of illustrations, and renders deterministic HTML/CSS into a print-ready PDF. Success means the user can understand generation progress, preview the final page style, choose visual assets confidently, and export a manuscript that feels intentionally designed.

## Brand Personality

Crafted, literary, practical. The interface should carry medieval manuscript atmosphere without turning controls into theatrical props. The generated document is the star; the app should feel calm, capable, and precise.

## Anti-references

Avoid generic SaaS dashboards, loud fantasy game menus, beige pages that look like flat CSS fills, default browser controls presented as finished design, pasted square illustrations, decorative frames in exported PDFs, and settings that require guessing from filenames.

## Design Principles

1. Show the manuscript, not just the controls.
2. Make asset choices visual and reversible.
3. Keep the interface readable while preserving medieval tone.
4. Separate app chrome from exported PDF styling.
5. Treat long documents as the default case.

## Accessibility & Inclusion

Target readable contrast for body text and controls. Preserve keyboard access for form fields, settings, file upload, preview actions, and language switching. Avoid motion that blocks task flow; any decorative transitions should remain subtle and nonessential.
